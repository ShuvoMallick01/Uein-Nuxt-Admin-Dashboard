Folder Structure

Folder structure

Finding what you need is made simpler by a clear and simple folder structure. The folder structure can be found in the directory Uno-dashboard

  • assets/: Stores all static assets like images (JPGs, PNGs, SVGs), fonts, and CSS files. These are processed and optimized by Vite during the build process in Nuxt.
  • components/: Contains reusable UI components, including those from ShadCN along with custom components designed for the template.
  • composables/: Holds global composables like (e.g., useApexChartOptions) for reusable logic throughout the app.
  • layout/: Contains the default layout as well as additional layouts used for different pages and sections across the application, providing flexibility in the overall site structure.
  • pages/: Includes all the pages of the application, each representing a distinct view or route.
  • plugins/: Contains any external or custom plugins integrated into the application, enhancing functionality.
  • public/: Stores publicly accessible static files like images, icons, and documents that donâ€™t require processing.
  • sections/: Holds sections for different pages (e.g., course, analytics etc). Sections include pre-built components and hooks specific to the page's functionality.
  • server/: Contains mock APIs and mock data for users, products, and invoices. It is used for local development and testing purposes, helping to simulate server-side data interactions in Nuxt 3.
  • stores/: Stores global state management using Pinia, for managing app-wide data and states.
  • types/: Contains TypeScript type definitions for better type safety throughout the project.
  • utils/: Stores utility functions and helper methods used across the application for better maintainability.
